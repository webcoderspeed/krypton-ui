---
title: Alert Dialog
description: A modal dialog that interrupts the user with important content and expects a response.
component: true
---

```tsx
<AlertDialog>
  <AlertDialogTrigger asChild>
    <Button variant="outline">Show Dialog</Button>
  </AlertDialogTrigger>
  <AlertDialogContent>
    <AlertDialogHeader>
      <AlertDialogTitle>Are you absolutely sure?</AlertDialogTitle>
      <AlertDialogDescription>
        This action cannot be undone. This will permanently delete your
        account and remove your data from our servers.
      </AlertDialogDescription>
    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel>Cancel</AlertDialogCancel>
      <AlertDialogAction>Continue</AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>
```

## Installation

```bash
npx shadcn@latest add alert-dialog
```


| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The content of the alert dialog |
| `defaultOpen` | `boolean` | `false` | The open state of the dialog when it is initially rendered. |
| `open` | `boolean` | `undefined` | The controlled open state of the dialog. |
| `onOpenChange` | `(open: boolean) => void` | `undefined` | Event handler called when the open state changes. |

## Styling

The alert dialog component uses Tailwind CSS classes for styling. You can customize the appearance by:

1. **Modifying the component**: Edit the alert dialog component file to change default styles
2. **Using className prop**: Pass additional Tailwind classes to any alert dialog component
3. **CSS variables**: Customize using CSS custom properties

### Action Button Variants

```tsx
// Default action button
<AlertDialogAction>Continue</AlertDialogAction>

// Destructive action button
<AlertDialogAction className="bg-destructive text-destructive-foreground hover:bg-destructive/90">
  Delete
</AlertDialogAction>

// Success action button
```

## Accessibility

The alert dialog component follows WAI-ARIA guidelines:

- Uses proper ARIA attributes for screen readers
- Supports keyboard navigation
- Traps focus within the dialog when open
- Provides focus management when opening and closing
- Supports escape key to close
- Cannot be dismissed by clicking outside (by design)
- Requires explicit user action (Cancel or Action button)

### Keyboard Interactions

| Key | Description |
| --- | --- |
| `Space` | Opens the alert dialog when focus is on the trigger |
| `Enter` | Opens the alert dialog when focus is on the trigger |
| `Tab` | Moves focus to the next focusable element within the dialog |
| `Shift + Tab` | Moves focus to the previous focusable element within the dialog |
| `Escape` | Closes the dialog and moves focus to the trigger |

## Best Practices

### Do's

- Use alert dialogs for critical actions that require user confirmation
- Always provide both Cancel and Action buttons
- Use clear, descriptive titles and descriptions
- Use destructive styling for dangerous actions
- Keep content concise and focused
- Provide clear action button labels

### Don'ts

- Don't use alert dialogs for non-critical information
- Don't use alert dialogs for complex forms or workflows
- Don't make alert dialogs dismissible by clicking outside
- Don't use vague button labels like "OK" or "Yes"
- Don't nest alert dialogs within other dialogs
- Don't use alert dialogs for success messages (use toast instead)

### When to Use Alert Dialog vs Dialog

**Use Alert Dialog when:**
- The action is destructive or irreversible
- User confirmation is required
- The action has significant consequences
- You need to interrupt the user's workflow

**Use Dialog when:**
- Collecting user input
- Displaying detailed information
- Creating or editing content
- The action is not destructive
