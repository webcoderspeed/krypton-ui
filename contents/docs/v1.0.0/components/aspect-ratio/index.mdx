## API Reference

### AspectRatio

Built on top of [Radix AspectRatio](https://www.radix-ui.com/docs/primitives/components/aspect-ratio).

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `ratio` | `number` | `1` | The desired aspect ratio (width / height). |
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child. |
| `className` | `string` | - | Additional CSS classes to apply to the container. |

## Common Aspect Ratios

| Ratio | Value | Use Case |
| --- | --- | --- |
| 16:9 | `16 / 9` | Widescreen videos, modern displays |
| 4:3 | `4 / 3` | Traditional TV, older displays |
| 1:1 | `1` | Square images, profile pictures |
| 3:2 | `3 / 2` | Photography, print media |
| 21:9 | `21 / 9` | Ultra-wide displays, cinematic |
| 9:16 | `9 / 16` | Mobile screens, stories |

## Best Practices

- Use consistent aspect ratios across similar content types
- Consider responsive behavior for different screen sizes
- Ensure images have appropriate `object-fit` properties
- Test with various content sizes and types
- Use semantic HTML elements when appropriate
