# Avatar

An image element with a fallback for representing the user. Built on top of Radix UI Avatar primitive.

## Installation

```bash
npm install @radix-ui/react-avatar
```

## Import

```tsx
import {
  Avatar,
  AvatarFallback,
  AvatarImage,
} from "@/components/ui/avatar"
```

## Usage

### Basic Example

```tsx
import {
  Avatar,
  AvatarFallback,
  AvatarImage,
} from "@/components/ui/avatar"

export function AvatarDemo() {
  return (
    <Avatar>
      <AvatarImage src="https://github.com/shadcn.png" alt="@shadcn" />
      <AvatarFallback>CN</AvatarFallback>
    </Avatar>
  )
}
```

## API Reference

### Avatar

The root avatar component that contains the image and fallback.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | The avatar image and fallback components |

**Default Styles:**
- `h-10 w-10` - 40px Ã— 40px size
- `rounded-full` - Circular shape
- `overflow-hidden` - Clips content to shape
- `shrink-0` - Prevents shrinking in flex containers

### AvatarImage

The image element of the avatar.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string` | - | The image source URL |
| `alt` | `string` | - | Alternative text for the image |
| `className` | `string` | - | Additional CSS classes |
| `onLoadingStatusChange` | `(status: "idle" \| "loading" \| "loaded" \| "error") => void` | - | Callback fired when loading status changes |

**Default Styles:**
- `aspect-square` - Maintains 1:1 aspect ratio
- `h-full w-full` - Fills the avatar container

### AvatarFallback

The fallback element displayed when the image fails to load or is loading.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | The fallback content (usually initials) |
| `delayMs` | `number` | - | Delay before showing fallback |

**Default Styles:**
- `flex h-full w-full` - Fills container and enables flexbox
- `items-center justify-center` - Centers content
- `rounded-full` - Circular shape
- `bg-muted` - Muted background color

## Features

- **Automatic Fallback**: Shows fallback when image fails to load
- **Accessible**: Proper alt text and ARIA attributes
- **Customizable**: Easy to style and resize
- **Loading States**: Built-in loading state management
- **Flexible**: Works with any image source
- **Responsive**: Scales well across different screen sizes

## Accessibility

- Always provide meaningful `alt` text for images
- Use appropriate contrast for fallback text
- Consider screen reader users when choosing fallback content
- Ensure sufficient color contrast for status indicators

## Styling

The component uses Tailwind CSS classes and can be customized by:

1. **Size**: Use `h-{size} w-{size}` classes
2. **Shape**: Use `rounded-{variant}` classes
3. **Border**: Add `border` and `ring` classes
4. **Background**: Customize fallback background with `bg-{color}` classes

## Common Patterns

### User Profile

```tsx
function UserProfile({ user }) {
  return (
    <div className="flex items-center space-x-3">
      <Avatar>
        <AvatarImage src={user.avatar} alt={user.name} />
        <AvatarFallback>
          {user.name.split(' ').map(n => n[0]).join('')}
        </AvatarFallback>
      </Avatar>
      <div>
        <p className="font-medium">{user.name}</p>
        <p className="text-sm text-muted-foreground">{user.email}</p>
      </div>
    </div>
  )
}
```

## Best Practices

1. **Always provide fallback**: Include meaningful initials or icons
2. **Use appropriate sizes**: Match avatar size to context (small for comments, large for profiles)
3. **Consistent styling**: Maintain consistent avatar styling across your app
4. **Meaningful alt text**: Provide descriptive alt text for images
5. **Handle loading states**: Consider showing loading indicators for slow connections
6. **Optimize images**: Use appropriately sized images to improve performance
7. **Consider accessibility**: Ensure sufficient contrast and screen reader support

## Troubleshooting

### Image not loading
- Check if the image URL is accessible
- Verify CORS settings for external images
- Ensure the fallback is properly configured

### Fallback not showing
- Make sure AvatarFallback is included as a child
- Check if there are any CSS conflicts
- Verify the image src is actually failing to load

### Size issues
- Use consistent width and height classes
- Adjust fallback text size accordingly
- Consider using aspect-ratio utilities for complex layouts
