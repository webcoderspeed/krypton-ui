# Badge

A versatile badge component for displaying status, notifications, or labels. Perfect for indicating counts, status updates, or categorizing content.

## Installation

The Badge component is included with the base installation.

## Import

```tsx
import { Badge } from 'krypton-ui'
```

## Basic Usage

The Badge component provides a simple way to display status or count information.

## API Reference

### Badge

The main badge component.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| variant | `"default" | "secondary" | "destructive" | "outline"` | `"default"` | The visual style variant |
| className | `string` | - | Additional CSS classes |
| children | `ReactNode` | - | The content to display |

## Features

- **Multiple Variants**: Default, secondary, destructive, and outline styles
- **Flexible Content**: Support for text, icons, and custom content
- **Customizable**: Easy to style with additional CSS classes
- **Accessible**: Proper semantic markup and ARIA support
- **Responsive**: Works well across different screen sizes

## Accessibility

- Uses semantic HTML elements
- Supports screen readers with proper text content
- Maintains good color contrast ratios
- Keyboard navigation friendly

## Best Practices

1. **Use appropriate variants** for different types of information
2. **Keep text concise** - badges work best with short labels
3. **Consider color meaning** - use destructive for errors, secondary for neutral info
4. **Maintain consistency** in badge usage across your application
5. **Test accessibility** with screen readers and keyboard navigation
6. **Use semantic colors** that align with your design system

## Troubleshooting

### Common Issues

1. **Badge not visible**: Check if the text color contrasts well with the background
2. **Layout issues**: Ensure proper spacing and alignment with flex utilities
3. **Overflow text**: Use appropriate sizing and consider truncation for long text
4. **Accessibility warnings**: Ensure badges have meaningful text content

### Styling Tips

- Use `hover:` variants sparingly as badges are typically not interactive
- Consider using `inline-flex` for better alignment with text
- Use consistent sizing across similar badge types
- Test color combinations for accessibility compliance
