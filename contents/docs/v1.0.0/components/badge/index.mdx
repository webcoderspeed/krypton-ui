## Installation

```bash
npx shadcn@latest add badge
```

## Usage

```tsx
import { Badge } from "@/components/ui/badge"
```

```tsx
<Badge>Badge</Badge>
```

## API Reference

### Badge

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `variant` | `"default" \| "secondary" \| "destructive" \| "outline"` | `"default"` | The visual style variant of the badge |
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `React.ReactNode` | - | The content to display inside the badge |

### Badge Variants

- **default**: Primary badge with solid background
- **secondary**: Muted badge with secondary colors
- **destructive**: Red badge for errors or warnings
- **outline**: Badge with border and transparent background

## Accessibility

- Badge uses semantic HTML with proper ARIA attributes
- Supports keyboard navigation and focus management
- Color combinations meet WCAG contrast requirements
- Screen reader friendly with proper text content

## Best Practices

1. **Keep text short**: Badges work best with 1-3 words
2. **Use appropriate variants**: Choose variants that match the semantic meaning
3. **Consider context**: Ensure badges don't interfere with the main content
4. **Consistent sizing**: Use consistent badge sizes within the same interface
5. **Meaningful colors**: Use colors that convey the right information to users

## Common Use Cases

- **Status indicators**: Show the current state of items (active, pending, completed)
- **Categories**: Label content with categories or tags
- **Notifications**: Display counts or new item indicators
- **Labels**: Add descriptive labels to content
- **Metadata**: Show additional information about items
