## API Reference

### Props

The Button component accepts all standard HTML button attributes plus:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" | "destructive" | "outline" | "secondary" | "ghost" | "link"` | `"default"` | The visual style variant of the button |
| `size` | `"default" | "sm" | "lg" | "icon"` | `"default"` | The size of the button |
| `asChild` | `boolean` | `false` | When true, the button will render as its child element |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The content of the button |

### Keyboard Navigation

- **Space** or **Enter**: Activates the button
- **Tab**: Moves focus to the next focusable element
- **Shift + Tab**: Moves focus to the previous focusable element

## Accessibility

The Button component follows WAI-ARIA guidelines:

- Uses semantic `<button>` element by default
- Supports keyboard navigation
- Provides proper focus management
- Includes appropriate ARIA attributes when disabled
- Maintains color contrast ratios for accessibility

## Best Practices

1. **Use descriptive labels**: Button text should clearly indicate what action will be performed
2. **Choose appropriate variants**: Use destructive for harmful actions, outline for secondary actions
3. **Consider loading states**: Show loading indicators for async operations
4. **Maintain consistency**: Use the same variant and size for similar actions throughout your app
5. **Provide adequate spacing**: Ensure buttons have enough space around them for easy interaction
