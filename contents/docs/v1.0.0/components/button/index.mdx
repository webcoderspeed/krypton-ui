# Button

A flexible and accessible button component that supports multiple variants, sizes, and states. Built on top of shadcn/ui with full TypeScript support and customizable styling.

## Installation

```bash
npm install krypton-ui
```

## Import

```tsx
import { Button } from 'krypton-ui'
```

## Basic Usage

The Button component provides a clean and modern interface for user interactions.

## API Reference

### Button Props

The Button component accepts all standard HTML button attributes plus:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'destructive' \| 'outline' \| 'secondary' \| 'ghost' \| 'link'` | `'default'` | The visual variant of the button |
| `size` | `'default' \| 'sm' \| 'lg' \| 'icon' \| 'xs'` | `'default'` | The size of the button |
| `asChild` | `boolean` | `false` | When true, renders as a Slot component for composition |
| `className` | `string` | - | Additional CSS classes to apply |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `onClick` | `(event: MouseEvent) => void` | - | Click event handler |

### Variants

- **default**: Primary button with solid background
- **destructive**: Red-themed button for dangerous actions
- **outline**: Button with border and transparent background
- **secondary**: Muted button for secondary actions
- **ghost**: Minimal button with hover effects
- **link**: Text-only button styled as a link

### Sizes

- **xs**: Extra small button (h-7)
- **sm**: Small button (h-9)
- **default**: Standard button (h-10)
- **lg**: Large button (h-11)
- **icon**: Square button for icons (h-9 w-9)

## Accessibility

The Button component follows WAI-ARIA guidelines:

- Supports keyboard navigation with Enter and Space keys
- Proper focus management with visible focus indicators
- Screen reader compatible with semantic button element
- Disabled state properly communicated to assistive technologies
- Support for `aria-label`, `aria-describedby`, and other ARIA attributes
