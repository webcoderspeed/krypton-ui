## API Reference

### Card

The root card container component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The content of the card |

### CardHeader

Container for the card's header content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The header content |

### CardTitle

The main title of the card.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The title content |

### CardDescription

A description or subtitle for the card.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The description content |

### CardContent

The main content area of the card.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The main content |

### CardFooter

Container for the card's footer content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The footer content |

## Accessibility

The Card component follows accessibility best practices:

- Uses semantic HTML structure
- Maintains proper heading hierarchy
- Supports keyboard navigation when interactive
- Provides adequate color contrast
- Works well with screen readers

## Best Practices

1. **Use semantic structure**: Include CardHeader, CardContent, and CardFooter for better organization
2. **Maintain consistency**: Use consistent spacing and styling across cards
3. **Consider responsive design**: Ensure cards work well on different screen sizes
4. **Limit content**: Keep card content concise and focused
5. **Use appropriate actions**: Place primary actions in the footer
