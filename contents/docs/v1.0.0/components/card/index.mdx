# Card

Displays a card with header, content, and footer. A flexible container for grouping related content and actions.

## Installation

No additional dependencies required.

## Import

```tsx
import {
  Card,
  CardContent,
  CardDescription,
  CardFooter,
  CardHeader,
  CardTitle,
} from "@/components/ui/card"
```

## API Reference

### Card

The root card container.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | The card content |

**Default Styles:**
- `rounded-lg` - Rounded corners
- `border` - Border styling
- `bg-card` - Card background color
- `text-card-foreground` - Text color
- `shadow-sm` - Subtle shadow

### CardHeader

Container for card title and description.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Header content |

**Default Styles:**
- `flex flex-col space-y-1.5` - Vertical layout with spacing
- `p-6` - Padding

### CardTitle

The main title of the card.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Title content |

**Default Styles:**
- `text-2xl font-semibold` - Large, bold text
- `leading-none tracking-tight` - Tight spacing

### CardDescription

Subtitle or description text.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Description content |

**Default Styles:**
- `text-sm text-muted-foreground` - Small, muted text

### CardContent

Main content area of the card.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Content |

**Default Styles:**
- `p-6 pt-0` - Padding (no top padding)

### CardFooter

Footer area for actions and additional content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Footer content |

**Default Styles:**
- `flex items-center` - Horizontal layout
- `p-6 pt-0` - Padding (no top padding)

## Features

- **Flexible Layout**: Compose cards with different combinations of components
- **Semantic Structure**: Clear hierarchy with header, content, and footer
- **Customizable**: Easy to style and extend
- **Responsive**: Works well across different screen sizes
- **Accessible**: Proper semantic HTML structure
- **Lightweight**: No external dependencies

## Accessibility

- Use semantic HTML elements when appropriate
- Ensure sufficient color contrast for text
- Add proper ARIA labels for interactive elements
- Consider keyboard navigation for interactive cards
- Use headings appropriately for card titles

## Common Patterns

### Card Grid

```tsx

  return (
    <div className="grid gap-6 md:grid-cols-2 lg:grid-cols-3">
```

## Best Practices

1. **Consistent Structure**: Use the same card structure across your application
2. **Appropriate Content**: Don't overcrowd cards with too much information
3. **Visual Hierarchy**: Use CardTitle and CardDescription to establish clear hierarchy
4. **Responsive Design**: Ensure cards work well on different screen sizes
5. **Loading States**: Consider loading states for dynamic card content
6. **Error Handling**: Handle error states gracefully within cards
7. **Performance**: Optimize images and content for better performance
8. **Accessibility**: Ensure cards are accessible to all users

## Troubleshooting

### Card not displaying correctly
- Check if the component is properly imported
- Verify that Tailwind CSS classes are being applied
- Ensure the parent container has appropriate sizing

### Spacing issues
- Use the default CardHeader, CardContent, and CardFooter components for consistent spacing
- Avoid adding extra padding that conflicts with default styles
- Use the `className` prop to override spacing when needed

### Content overflow
- Set appropriate width constraints on the Card component
- Use `overflow-hidden` class when necessary
- Consider responsive design for different screen sizes
