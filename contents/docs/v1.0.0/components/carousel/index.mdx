### With API

Access carousel API for programmatic control.

<CodePreview 
  code={`import {
  Carousel,
  CarouselContent,
  CarouselItem,
  CarouselNext,
  CarouselPrevious,
} from '@/components/ui/carousel'
import { Card, CardContent } from '@/components/ui/card'
import { Button } from '@/components/ui/button'
import { useEffect, useState } from 'react'
import type { CarouselApi } from '@/components/ui/carousel'

export default function CarouselApi() {
  const [api, setApi] = useState<CarouselApi>()
  const [current, setCurrent] = useState(0)
  const [count, setCount] = useState(0)

  useEffect(() => {
    if (!api) {
      return
    }

    setCount(api.scrollSnapList().length)
    setCurrent(api.selectedScrollSnap() + 1)

    api.on("select", () => {
      setCurrent(api.selectedScrollSnap() + 1)
    })
  }, [api])

  return (
    <div className="mx-auto max-w-xs">
      <Carousel setApi={setApi} className="w-full max-w-xs">
        <CarouselContent>
          {Array.from({ length: 5 }).map((_, index) => (
            <CarouselItem key={index}>
              <Card>
                <CardContent className="flex aspect-square items-center justify-center p-6">
                  <span className="text-4xl font-semibold">{index + 1}</span>
                </CardContent>
              </Card>
            </CarouselItem>
          ))}
        </CarouselContent>
        <CarouselPrevious />
        <CarouselNext />
      </Carousel>
      <div className="py-2 text-center text-sm text-muted-foreground">
        Slide {current} of {count}
      </div>
      <div className="flex justify-center gap-2 mt-2">
        <Button
          variant="outline"
          size="sm"
          onClick={() => api?.scrollPrev()}
        >
          Previous
        </Button>
        <Button
          variant="outline"
          size="sm"
          onClick={() => api?.scrollNext()}
        >
          Next
        </Button>
      </div>
    </div>
  )
}`} 
  language="tsx" 
  preview={<CarouselApi />} 
/>

## API Reference

### Carousel

The root carousel container component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"horizontal" | "vertical"` | `"horizontal"` | The orientation of the carousel |
| `opts` | `EmblaOptionsType` | - | Options to pass to the underlying Embla carousel |
| `setApi` | `(api: CarouselApi) => void` | - | Callback to get the carousel API instance |
| `plugins` | `EmblaPluginType[]` | - | Plugins to add to the carousel |
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The carousel content |

### CarouselContent

Container for carousel items.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The carousel items |

### CarouselItem

Individual carousel item wrapper.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The item content |

### CarouselPrevious

Previous navigation button.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `variant` | `ButtonVariant` | `"outline"` | Button variant |
| `size` | `ButtonSize` | `"icon"` | Button size |

### CarouselNext

Next navigation button.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `variant` | `ButtonVariant` | `"outline"` | Button variant |
| `size` | `ButtonSize` | `"icon"` | Button size |

### Keyboard Navigation

- **Arrow Keys**: Navigate between slides (when focused)
- **Tab**: Move focus to navigation buttons
- **Enter** or **Space**: Activate focused navigation button

## Accessibility

The Carousel component follows accessibility best practices:

- Uses semantic HTML with proper ARIA labels
- Supports keyboard navigation
- Provides screen reader announcements
- Maintains focus management
- Includes proper role and state attributes
- Respects reduced motion preferences

## Best Practices

1. **Provide navigation**: Always include previous/next buttons for accessibility
2. **Consider autoplay carefully**: Use autoplay sparingly and provide pause controls
3. **Optimize for touch**: Ensure smooth touch/swipe interactions on mobile
4. **Limit content**: Keep carousel items focused and not overwhelming
5. **Provide indicators**: Show current position for longer carousels
