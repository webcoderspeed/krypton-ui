---
title: Dialog
description: A window overlaid on either the primary window or another dialog window, rendering the content underneath inert.
component: true
---

```tsx
<Dialog>
  <DialogTrigger asChild>
    <Button variant="outline">Edit Profile</Button>
  </DialogTrigger>
  <DialogContent className="sm:max-w-[425px]">
    <DialogHeader>
      <DialogTitle>Edit profile</DialogTitle>
      <DialogDescription>
        Make changes to your profile here. Click save when you're done.
      </DialogDescription>
    </DialogHeader>
    <div className="grid gap-4 py-4">
      <div className="grid grid-cols-4 items-center gap-4">
        <Label htmlFor="name" className="text-right">
          Name
        </Label>
        <Input id="name" value="Pedro Duarte" className="col-span-3" />
      </div>
      <div className="grid grid-cols-4 items-center gap-4">
        <Label htmlFor="username" className="text-right">
          Username
        </Label>
        <Input id="username" value="@peduarte" className="col-span-3" />
      </div>
    </div>
    <DialogFooter>
      <Button type="submit">Save changes</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

## Installation

```bash
npx shadcn@latest add dialog
```

## Usage

```tsx
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from "@/components/ui/dialog"
```

```tsx
<Dialog>
  <DialogTrigger>Open</DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Are you absolutely sure?</DialogTitle>
      <DialogDescription>
        This action cannot be undone. This will permanently delete your account
        and remove your data from our servers.
      </DialogDescription>
    </DialogHeader>
  </DialogContent>
</Dialog>
```

## API Reference

### Dialog

Built on top of [Radix Dialog](https://www.radix-ui.com/docs/primitives/components/dialog).

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The content of the dialog |
| `defaultOpen` | `boolean` | `false` | The open state of the dialog when it is initially rendered. |
| `open` | `boolean` | `undefined` | The controlled open state of the dialog. |
| `onOpenChange` | `(open: boolean) => void` | `undefined` | Event handler called when the open state changes. |
| `modal` | `boolean` | `true` | When true, focus will be trapped within the dialog. |

### DialogTrigger

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The trigger element |
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child. |

### DialogContent

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The content of the dialog |
| `className` | `string` | `undefined` | Additional CSS classes to apply to the dialog content. |
| `onOpenAutoFocus` | `(event: Event) => void` | `undefined` | Event handler called when focus moves into the component. |
| `onCloseAutoFocus` | `(event: Event) => void` | `undefined` | Event handler called when focus moves to the trigger after closing. |
| `onEscapeKeyDown` | `(event: KeyboardEvent) => void` | `undefined` | Event handler called when the escape key is down. |
| `onPointerDownOutside` | `(event: PointerDownOutsideEvent) => void` | `undefined` | Event handler called when a pointer event occurs outside the bounds. |
| `onInteractOutside` | `(event: FocusOutsideEvent \| PointerDownOutsideEvent) => void` | `undefined` | Event handler called when an interaction happens outside the component. |

### DialogHeader

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The header content |
| `className` | `string` | `undefined` | Additional CSS classes to apply to the dialog header. |

### DialogFooter

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The footer content |
| `className` | `string` | `undefined` | Additional CSS classes to apply to the dialog footer. |

### DialogTitle

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The title content |
| `className` | `string` | `undefined` | Additional CSS classes to apply to the dialog title. |

### DialogDescription

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The description content |
| `className` | `string` | `undefined` | Additional CSS classes to apply to the dialog description. |

### DialogClose

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The close trigger element |
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child. |

## Accessibility

The dialog component follows WAI-ARIA guidelines:

- Uses proper ARIA attributes for screen readers
- Supports keyboard navigation
- Traps focus within the dialog when open
- Provides focus management when opening and closing
- Supports escape key to close
- Clicking outside closes the dialog (can be disabled)

### Keyboard Interactions

| Key | Description |
| --- | --- |
| `Space` | Opens/closes the dialog when focus is on the trigger |
| `Enter` | Opens/closes the dialog when focus is on the trigger |
| `Tab` | Moves focus to the next focusable element within the dialog |
| `Shift + Tab` | Moves focus to the previous focusable element within the dialog |
| `Escape` | Closes the dialog and moves focus to the trigger |

## Best Practices

### Do&apos;s

- Always provide a clear title and description
- Use dialogs for important actions that require user attention
- Keep dialog content focused and concise
- Provide clear action buttons (Save, Cancel, etc.)
- Use appropriate dialog sizes for content
- Include proper focus management

### Don&apos;ts

- Don&apos;t use dialogs for non-critical information
- Don&apos;t make dialogs too large or too small for their content
- Don&apos;t nest dialogs within other dialogs
- Don&apos;t use dialogs for complex workflows (use pages instead)
- Don&apos;t forget to handle loading and error states

### Form Integration Best Practices

```tsx
function FormDialog() {
  const [open, setOpen] = useState(false)
  const [loading, setLoading] = useState(false)

  const handleSubmit = async (data: FormData) => {
    setLoading(true)
    try {
      await submitForm(data)
      setOpen(false) // Close dialog on success
    } catch (error) {
      // Handle error
    } finally {
      setLoading(false)
    }
  }

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>Open Form</Button>
      </DialogTrigger>
      <DialogContent>
        <form onSubmit={handleSubmit}>
          <DialogHeader>
            <DialogTitle>Form Title</DialogTitle>
            <DialogDescription>
              Form description
            </DialogDescription>
          </DialogHeader>
          
          {/* Form fields */}
          
          <DialogFooter>
            <Button 
              type="button" 
              variant="outline" 
              onClick={() => setOpen(false)}
            >
              Cancel
            </Button>
            <Button type="submit" disabled={loading}>
              {loading ? "Saving..." : "Save"}
            </Button>
          </DialogFooter>
        </form>
      </DialogContent>
    </Dialog>
  )
}
```

## Advanced Usage

### Controlled Dialog

```tsx
function ControlledDialog() {
  const [open, setOpen] = useState(false)

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>Open Dialog</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Controlled Dialog</DialogTitle>
        </DialogHeader>
        <DialogFooter>
          <Button onClick={() => setOpen(false)}>
            Close
          </Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  )
}
```

### Custom Close Behavior


## Troubleshooting

### Common Issues

1. **Dialog not opening**
   - Check if the trigger is properly wrapped with DialogTrigger
   - Verify that the Dialog component wraps both trigger and content
   - Ensure no JavaScript errors are preventing the dialog from opening

2. **Focus issues**
   - Make sure focusable elements are properly structured
   - Check if custom focus management is interfering
   - Verify that the dialog content is properly rendered

3. **Styling problems**
   - Ensure Tailwind CSS is properly configured
   - Check if custom styles are conflicting with default styles
   - Verify z-index values for proper layering

4. **Form submission issues**
   - Make sure form elements are properly structured
   - Check if event handlers are correctly attached
   - Verify that form validation is working as expected

### Performance Considerations

- Use controlled dialogs sparingly to avoid unnecessary re-renders
- Consider lazy loading dialog content for better performance
- Avoid complex animations that might impact performance
- Use `React.memo` for dialog components that don&apos;t change frequently

```tsx
const OptimizedDialog = React.memo(({ children, ...props }) => {
  return (
    <Dialog {...props}>
      {children}
    </Dialog>
  )
})
```
