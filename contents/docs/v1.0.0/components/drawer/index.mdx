## Real-World Examples

### Shopping Cart

A comprehensive shopping cart drawer with item management and checkout functionality.

<CodePreview 
  code={`import {
  Drawer,
  DrawerClose,
  DrawerContent,
  DrawerDescription,
  DrawerFooter,
  DrawerHeader,
  DrawerTitle,
  DrawerTrigger,
} from "@/components/ui/drawer"
import { Button } from "@/components/ui/button"
import { Badge } from "@/components/ui/badge"
import { Separator } from "@/components/ui/separator"
import { ShoppingCart, Plus, Minus, X } from "lucide-react"

export default function DrawerShoppingCart() {
  const cartItems = [
    { id: 1, name: "Wireless Headphones", price: 99.99, quantity: 1, image: "/api/placeholder/60/60" },
    { id: 2, name: "Smart Watch", price: 299.99, quantity: 2, image: "/api/placeholder/60/60" },
    { id: 3, name: "Phone Case", price: 24.99, quantity: 1, image: "/api/placeholder/60/60" },
  ]

  const subtotal = cartItems.reduce((sum, item) => sum + (item.price * item.quantity), 0)
  const shipping = 9.99
  const tax = subtotal * 0.08
  const total = subtotal + shipping + tax

  return (
    <Drawer>
      <DrawerTrigger asChild>
        <Button variant="outline" className="relative">
          <ShoppingCart className="h-4 w-4 mr-2" />
          Cart
          <Badge variant="destructive" className="ml-2 h-5 w-5 rounded-full p-0 text-xs">
            {cartItems.reduce((sum, item) => sum + item.quantity, 0)}
          </Badge>
        </Button>
      </DrawerTrigger>
      <DrawerContent className="max-h-[90vh]">
        <DrawerHeader>
          <DrawerTitle>Shopping Cart</DrawerTitle>
          <DrawerDescription>
            Review your items and proceed to checkout
          </DrawerDescription>
        </DrawerHeader>
        <div className="p-4 overflow-y-auto">
          <div className="space-y-4">
            {cartItems.map((item) => (
              <div key={item.id} className="flex items-center gap-4 p-3 border rounded-lg">
                <div className="w-15 h-15 bg-muted rounded-md flex items-center justify-center">
                  <span className="text-xs">IMG</span>
                </div>
                <div className="flex-1">
                  <h4 className="font-medium">{item.name}</h4>
                  <p className="text-sm text-muted-foreground">\${item.price}</p>
                </div>
                <div className="flex items-center gap-2">
                  <Button variant="outline" size="icon" className="h-8 w-8">
                    <Minus className="h-3 w-3" />
                  </Button>
                  <span className="w-8 text-center">{item.quantity}</span>
                  <Button variant="outline" size="icon" className="h-8 w-8">
                    <Plus className="h-3 w-3" />
                  </Button>
                </div>
                <Button variant="ghost" size="icon" className="h-8 w-8">
                  <X className="h-3 w-3" />
                </Button>
              </div>
            ))}
          </div>
          <Separator className="my-4" />
          <div className="space-y-2">
            <div className="flex justify-between text-sm">
              <span>Subtotal</span>
              <span>\${subtotal.toFixed(2)}</span>
            </div>
            <div className="flex justify-between text-sm">
              <span>Shipping</span>
              <span>\${shipping.toFixed(2)}</span>
            </div>
            <div className="flex justify-between text-sm">
              <span>Tax</span>
              <span>\${tax.toFixed(2)}</span>
            </div>
            <Separator />
            <div className="flex justify-between font-medium">
              <span>Total</span>
              <span>\${total.toFixed(2)}</span>
            </div>
          </div>
        </div>
        <DrawerFooter>
          <Button className="w-full">Proceed to Checkout</Button>
          <DrawerClose asChild>
            <Button variant="outline">Continue Shopping</Button>
          </DrawerClose>
        </DrawerFooter>
      </DrawerContent>
    </Drawer>
  )
}`} 
  language="tsx" 
  preview={<DrawerShoppingCart />} 
/>

## API Reference

### Drawer

The root drawer component that manages the drawer state.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | The controlled open state of the drawer |
| `onOpenChange` | `(open: boolean) => void` | - | Event handler called when the open state changes |
| `shouldScaleBackground` | `boolean` | `true` | Whether the background should scale when the drawer is open |
| `modal` | `boolean` | `true` | Whether the drawer should be modal |
| `children` | `ReactNode` | - | The drawer content |

### DrawerTrigger

The element that triggers the drawer. Must be a single focusable element.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child |
| `children` | `ReactNode` | - | The element that triggers the drawer |

### DrawerContent

The container for the drawer content that slides in from the bottom.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The content of the drawer |

### DrawerHeader

The header section of the drawer, typically containing the title and description.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The header content |

### DrawerTitle

The title of the drawer. Important for accessibility.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The title content |

### DrawerDescription

The description of the drawer. Provides additional context.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The description content |

### DrawerFooter

The footer section of the drawer, typically containing action buttons.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The footer content |

### DrawerClose

A button that closes the drawer when clicked.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child |
| `children` | `ReactNode` | - | The close button content |

## Accessibility

The Drawer component follows WAI-ARIA guidelines and includes:

### Keyboard Support

| Key | Description |
|-----|-------------|
| `Escape` | Closes the drawer when it's open |
| `Tab` | Moves focus to the next focusable element within the drawer |
| `Shift + Tab` | Moves focus to the previous focusable element within the drawer |

### Screen Reader Support

- Uses proper ARIA attributes for drawer identification
- Drawer content is announced when opened
- Focus is trapped within the drawer when open
- Focus returns to the trigger element when closed
- Supports both mouse and keyboard interactions

### Focus Management

- Automatically focuses the drawer content when opened
- Traps focus within the drawer while open
- Returns focus to the trigger element when closed
- Supports custom focus management with proper ARIA attributes

## Best Practices

1. **Mobile-First Design**: Drawers are primarily designed for mobile interfaces
2. **Keep Content Focused**: Limit drawer content to essential information and actions
3. **Use Clear Headers**: Always include a descriptive title and optional description
4. **Provide Clear Actions**: Include obvious close buttons and primary action buttons
5. **Consider Height**: Ensure content fits comfortably within the drawer height
6. **Handle Scrolling**: Use scrollable content for long lists or forms
7. **Test on Devices**: Always test drawer interactions on actual mobile devices
8. **Backdrop Interaction**: Allow users to close the drawer by tapping the backdrop
