A popover is a non-modal dialog that floats around a trigger element. It's used to display additional information, forms, or actions without navigating away from the current context. Unlike tooltips, popovers can contain interactive content and remain open until explicitly closed.

## API Reference

### Popover

The root popover component that manages the open/closed state.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultOpen` | `boolean` | `false` | Whether the popover is open by default |
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when open state changes |
| `modal` | `boolean` | `false` | Whether the popover should be modal |

### PopoverTrigger

The button or element that triggers the popover. Must be passed as a child to `Popover`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Change the component to the HTML tag or custom component of the only child |

## Accessibility

The Popover component follows the [WAI-ARIA Popover Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/popover/) and includes:

- **Keyboard Navigation**: 
  - `Space` or `Enter` to open/close the popover
  - `Escape` to close the popover
  - Tab navigation within the popover content

- **Screen Reader Support**:
  - Proper ARIA attributes for popover identification
  - Focus management when opening/closing
  - Accessible trigger and content association

- **Focus Management**:
  - Focus returns to trigger when popover closes
  - Focus trap within popover when modal
  - Proper focus indicators

## Best Practices

1. **Content**: Keep popover content concise and focused on a single task or piece of information.

2. **Positioning**: Use appropriate positioning based on available space and content importance.

3. **Triggers**: Use clear, descriptive trigger elements that indicate interactive behavior.

4. **Responsive**: Consider mobile experiences where popovers might need different positioning or behavior.

5. **Performance**: Avoid heavy content in popovers that might cause layout shifts or performance issues.

6. **Accessibility**: Ensure all interactive elements within popovers are keyboard accessible and properly labeled.
