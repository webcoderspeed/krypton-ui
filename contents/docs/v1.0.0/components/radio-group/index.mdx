## API Reference

### RadioGroup

The root component that manages the radio group state and provides context to child components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | The controlled value of the radio group |
| `defaultValue` | `string` | - | The default value when uncontrolled |
| `onValueChange` | `(value: string) => void` | - | Callback fired when the value changes |
| `disabled` | `boolean` | `false` | Whether the entire radio group is disabled |
| `required` | `boolean` | `false` | Whether the radio group is required |
| `name` | `string` | - | The name attribute for the radio group |
| `className` | `string` | - | Additional CSS classes to apply |
| `orientation` | `"horizontal" \| "vertical"` | `"vertical"` | The orientation of the radio group |
| `dir` | `"ltr" \| "rtl"` | - | The reading direction |
| `loop` | `boolean` | `true` | Whether keyboard navigation should loop |

### RadioGroupItem

Individual radio button component within a RadioGroup.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | The value of this radio item (required) |
| `disabled` | `boolean` | `false` | Whether this specific item is disabled |
| `required` | `boolean` | `false` | Whether this item is required |
| `className` | `string` | - | Additional CSS classes to apply |
| `id` | `string` | - | The id attribute for the radio item |

## Best Practices

1. **Always use labels**: Associate each RadioGroupItem with a Label for accessibility
2. **Provide clear options**: Use descriptive labels that clearly differentiate options
3. **Consider default values**: Set sensible defaults for better user experience
4. **Group related options**: Only use radio groups for mutually exclusive choices
5. **Limit options**: Keep the number of options reasonable (typically 2-7)
6. **Use consistent styling**: Maintain visual consistency across your application
7. **Handle validation**: Provide clear feedback for required radio groups
8. **Consider mobile**: Ensure touch targets are large enough for mobile devices
