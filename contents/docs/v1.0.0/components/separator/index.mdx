## API Reference

### Separator

Built on top of [Radix Separator](https://www.radix-ui.com/docs/primitives/components/separator).

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | The orientation of the separator. |
| `decorative` | `boolean` | `true` | Whether the separator is decorative or semantic. |
| `className` | `string` | `-` | Additional CSS classes to apply to the separator. |

## Accessibility

### Keyboard Navigation

- The separator is not focusable and does not participate in keyboard navigation.
- When `decorative={false}`, the separator has semantic meaning and is announced by screen readers.

### Screen Reader Support

- By default, the separator is decorative (`decorative={true}`) and is ignored by screen readers.
- Set `decorative={false}` to make the separator semantic and announce it to screen readers.
- Use appropriate ARIA labels when the separator has semantic meaning.

### Best Practices

- Use horizontal separators to divide content sections vertically.
- Use vertical separators to divide inline content horizontally.
- Ensure sufficient color contrast for the separator line.
- Consider the semantic meaning when choosing between decorative and semantic separators.

## Styling

### Default Styles

The separator comes with default styles that can be customized:

- **Background**: Uses the `border` color from your theme
- **Horizontal**: 1px height, full width
- **Vertical**: 1px width, full height
- **Shrink**: Prevents the separator from growing

### Customization

```tsx
// Custom color
<Separator className="bg-primary" />

// Custom thickness
<Separator className="h-[2px]" />
<Separator orientation="vertical" className="w-[2px]" />

// Custom spacing
<Separator className="my-8" />

// Gradient separator
<Separator className="bg-gradient-to-r from-transparent via-border to-transparent" />

// Dashed style
<Separator className="border-dashed border-t border-border bg-transparent h-0" />
```

### Design Tokens

The separator uses the following design tokens:

- `border`: Default separator color
- `primary`: For accent separators
- `muted`: For subtle separators
- `destructive`: For error state separators
