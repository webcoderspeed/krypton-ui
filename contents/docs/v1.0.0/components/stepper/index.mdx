## API Reference

### Stepper

The main container component that manages the step layout and numbering.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes to apply to the stepper container |
| `children` | `ReactNode` | - | StepperItem components to display as steps |

### StepperItem

Individual step component that contains the step content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | The title/label for the step |
| `className` | `string` | - | Additional CSS classes to apply to the step item |
| `children` | `ReactNode` | - | The content to display for this step |

## Accessibility

The Stepper component follows accessibility best practices:

- **Semantic Structure**: Uses proper HTML structure with ordered lists and headings
- **Keyboard Navigation**: Supports keyboard navigation when interactive elements are included
- **Screen Reader Support**: Provides clear step numbering and titles for screen readers
- **Focus Management**: Maintains proper focus order through the steps
- **ARIA Labels**: Uses appropriate ARIA attributes for enhanced accessibility
- **Color Contrast**: Ensures adequate color contrast for step indicators and text

## Best Practices

1. **Clear Step Titles**: Use descriptive, concise titles that clearly indicate what each step involves
2. **Logical Flow**: Organize steps in a logical, sequential order that makes sense to users
3. **Consistent Content**: Maintain consistent formatting and content structure across all steps
4. **Progress Indication**: The numbered indicators help users understand their progress
5. **Responsive Design**: Ensure the stepper works well on different screen sizes
6. **Content Length**: Keep step content focused and avoid overwhelming users with too much information
7. **Visual Hierarchy**: Use proper heading levels and visual hierarchy within step content
8. **Error Handling**: If using interactive elements, provide clear error states and validation
