---
title: Tabs
description: A set of layered sections of content—known as tab panels—that are displayed one at a time.
---

Tabs organize content into multiple sections and allow users to navigate between them. Each tab panel is associated with a tab that serves as a label for the panel.

## Usage

```tsx
import { Tabs, TabsContent, TabsList, TabsTrigger } from "@/components/ui/tabs"
```

## API Reference

### Tabs

The root container for the tabs component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultValue` | `string` | - | The default active tab |
| `value` | `string` | - | The controlled active tab |
| `onValueChange` | `(value: string) => void` | - | Callback when active tab changes |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | The orientation of the tabs |
| `dir` | `"ltr" \| "rtl"` | `"ltr"` | The reading direction |
| `activationMode` | `"automatic" \| "manual"` | `"automatic"` | How tabs are activated |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Tab content |

### TabsList

Container for the tab triggers.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Tab triggers |

### TabsTrigger

Clickable element that activates a tab panel.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | The value that identifies this tab |
| `disabled` | `boolean` | `false` | Whether the tab is disabled |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Trigger content |

### TabsContent

Container for the tab panel content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | The value that identifies this panel |
| `forceMount` | `boolean` | `false` | Force mount the content |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Panel content |

## Accessibility

The Tabs component follows the [WAI-ARIA Tabs Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/):

- **Keyboard Navigation**: Arrow keys navigate between tabs, Space/Enter activates tabs
- **Focus Management**: Focus is properly managed when switching between tabs
- **ARIA Attributes**: Proper `role`, `aria-selected`, and `aria-controls` attributes
- **Screen Reader Support**: Tab panels are properly associated with their triggers

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` | Move focus to the next focusable element |
| `Shift + Tab` | Move focus to the previous focusable element |
| `ArrowRight` / `ArrowDown` | Move focus to the next tab |
| `ArrowLeft` / `ArrowUp` | Move focus to the previous tab |
| `Space` / `Enter` | Activate the focused tab |
| `Home` | Move focus to the first tab |
| `End` | Move focus to the last tab |

## Best Practices

1. **Logical Grouping**: Group related content that users might want to compare
2. **Clear Labels**: Use descriptive, concise labels for tab triggers
3. **Consistent Content**: Maintain similar content structure across tabs
4. **Reasonable Number**: Limit the number of tabs to avoid overwhelming users
5. **Default Selection**: Always have a default active tab
6. **Loading States**: Show loading indicators for tabs with async content
7. **Responsive Design**: Consider how tabs behave on smaller screens
8. **Disabled States**: Clearly indicate when tabs are disabled and why
