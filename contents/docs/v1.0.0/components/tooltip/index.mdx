## Usage

### Default

A basic tooltip with simple text content.

<CodePreview 
  code={`import {
  Tooltip,
  TooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from "@/components/ui/tooltip"
import { Button } from "@/components/ui/button"

export default function TooltipDefault() {
  return (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger asChild>
          <Button variant="outline">Hover me</Button>
        </TooltipTrigger>
        <TooltipContent>
          <p>Add to library</p>
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  )
}`} 
  language="tsx" 
  preview={<TooltipDefault />} 
/>

### Delays

Configure tooltip delays for better user experience.

<CodePreview 
  code={`import {
  Tooltip,
  TooltipContent,

### Data Visualization Tooltips

  TooltipProvider,
  TooltipTrigger,
} from "@/components/ui/tooltip"

import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card"
import { TrendingUp, TrendingDown, Minus } from "lucide-react"

export default function TooltipDataVisualization() {
  const chartData = [
    { month: "Jan", value: 65, change: 12 },
    { month: "Feb", value: 78, change: 8 },
    { month: "Mar", value: 52, change: -15 },
    { month: "Apr", value: 91, change: 25 },
    { month: "May", value: 87, change: -2 },
    { month: "Jun", value: 95, change: 18 },
  ]

  const metrics = [
    { label: "Conversion Rate", value: 3.2, target: 4.0, trend: "up" },
    { label: "Bounce Rate", value: 42.1, target: 35.0, trend: "down" },
    { label: "Session Duration", value: 2.8, target: 3.0, trend: "neutral" },
  ]

  return (
    <TooltipProvider>
      <div className="space-y-6">
        <Card>
          <CardHeader>
            <CardTitle>Monthly Revenue</CardTitle>
          </CardHeader>
          <CardContent>
            <div className="flex items-end gap-2 h-32">
              {chartData.map((data, index) => (
                <Tooltip key={data.month}>
                  <TooltipTrigger asChild>
                    <div className="flex flex-col items-center gap-1 cursor-help">
                      <div 
                        className="w-8 bg-blue-500 rounded-t hover:bg-blue-600 transition-colors"
                        style={{ height: \`\${data.value}%\` }}
                      />
                      <span className="text-xs text-muted-foreground">{data.month}</span>
                    </div>
                  </TooltipTrigger>
                  <TooltipContent>
                    <div className="space-y-1">
                      <p className="font-medium">{data.month} 2024</p>
                      <p className="text-sm">Revenue: $\{data.value}K</p>
                      <p className="text-sm flex items-center gap-1">
                        Change: 
                        {data.change > 0 ? (
                          <span className="text-green-500 flex items-center">
                            <TrendingUp className="h-3 w-3" />
                            +{data.change}%

                        ) : data.change < 0 ? (

## API Reference

### TooltipProvider

Provides context for all tooltip components. Wrap your app or component tree with this provider.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `delayDuration` | `number` | `700` | The duration from when the mouse enters a tooltip trigger until the tooltip opens (in milliseconds) |
| `skipDelayDuration` | `number` | `300` | How much time a user has to enter another trigger without incurring a delay again (in milliseconds) |
| `disableHoverableContent` | `boolean` | `false` | When true, trying to hover the content will result in the tooltip closing |
| `children` | `ReactNode` | - | The content that can trigger tooltips |

### Tooltip

The root tooltip component that manages the tooltip state.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | The controlled open state of the tooltip |
| `defaultOpen` | `boolean` | `false` | The open state of the tooltip when it is initially rendered |
| `onOpenChange` | `(open: boolean) => void` | - | Event handler called when the open state changes |
| `delayDuration` | `number` | - | Override the duration given to the Provider to customize for this tooltip |
| `disableHoverableContent` | `boolean` | - | Override the disableHoverableContent given to the Provider |

### TooltipTrigger

The element that triggers the tooltip. Must be a single focusable element.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child |
| `children` | `ReactNode` | - | The element that triggers the tooltip |

### TooltipContent

The component that contains the tooltip content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"top"` | The preferred side of the trigger to render against |
| `sideOffset` | `number` | `4` | The distance in pixels from the trigger |
| `align` | `"start" \| "center" \| "end"` | `"center"` | The preferred alignment against the trigger |
| `alignOffset` | `number` | `0` | An offset in pixels from the "start" or "end" alignment options |
| `avoidCollisions` | `boolean` | `true` | When true, overrides the side and align preferences to prevent collisions |
| `collisionBoundary` | `Element \| null \| Array<Element \| null>` | `[]` | The element used as the collision boundary |
| `collisionPadding` | `number \| Partial<Record<Side, number>>` | `0` | The distance in pixels from the boundary edges where collision detection should occur |
| `arrowPadding` | `number` | `0` | The padding between the arrow and the edges of the content |
| `sticky` | `"partial" \| "always"` | `"partial"` | The sticky behavior on the align axis |
| `hideWhenDetached` | `boolean` | `false` | Whether to hide the content when the trigger becomes fully occluded |
| `className` | `string` | - | Additional CSS classes to apply |
| `children` | `ReactNode` | - | The content of the tooltip |

## Accessibility

The Tooltip component follows WAI-ARIA guidelines and includes:

### Keyboard Support

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the next focusable element |
| `Shift + Tab` | Moves focus to the previous focusable element |
| `Escape` | Closes the tooltip if it's open |

### Screen Reader Support

- Uses `aria-describedby` to associate tooltip content with trigger elements
- Tooltip content is announced when the trigger receives focus
- Supports both mouse and keyboard interactions
- Content is properly exposed to assistive technologies

### Focus Management

- Tooltips appear on both hover and focus
- Focus remains on the trigger element when tooltip is shown
- Tooltips automatically close when focus moves away
- Supports custom focusable elements with proper ARIA attributes

## Best Practices

1. **Keep content concise**: Tooltips should provide brief, helpful information
2. **Use for non-essential information**: Don't put critical information only in tooltips
3. **Ensure keyboard accessibility**: Always test with keyboard navigation
4. **Consider mobile users**: Tooltips may not work well on touch devices
5. **Provide adequate contrast**: Ensure tooltip content is readable
6. **Use appropriate delays**: Don't make users wait too long for helpful information
7. **Position thoughtfully**: Ensure tooltips don't cover important content
8. **Test with screen readers**: Verify that tooltip content is properly announced
